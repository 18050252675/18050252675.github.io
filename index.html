0613 集合异常

集合大家族

组长：collection

List 和Set的区别

  • Set 集 元素是无序的，不可重复的
  • List 列表 元素是有序的，可重复的
    • tip：有序和无序
      • 有序：有索引（下标），添加的顺序就是存储的顺序（按下标）
      • 无序：没有索引，添加的顺序不一定是存储的顺序（有自己的规则，哈希规则（hash））
    • ArrayList: 数组列表 底层的数据结构是数组（有下标，长度不可变，查询效率高，增删改查效率比较低）
    • LinkedList：链表列表 底层的数据结构是链表(查询效率比较低，但是增加删除的效率比较高)

存储数据的方式：

​ 数组、对象、集合

数组、对象和集合的区别

数组：长度固定、所以的数据类型都有对应的数组，同一个数组只能存同一种类型

对象： 属性的格式是固定的，可以定义类的类型，同一个对象可以存不同的类型

集合：长度可变，集合对数据进行了封装，所以的引用数据类型都要对应的集合，（接口，对象，类类型，）

基本数据类型没有集合，

  • ​ 每个基本数据类型都有对应的包装类（封装类）
    • int—》integer(拓展)
  • 同一个集合可以存不同的数据类型，但是不推荐！
  • 通常同一个集合都会规定同一种类型，这种规定叫做”泛型“
  • 定义泛型： <”数据类型”>： int类型的泛型

容器 增删改查

  • ​ 增加

    • 直接往末尾添加 add(元素)
    • 指定位置添加元素 add(下标，新增的元素)

  • 删(返回值为删除的对象)

    • 根据下标删除 remove(下标)
    • 根据对象删除 remove(对象名)
    • 清空 clear()

  • 改

    • 根据下标修改 set(下标，新元素)

  • 查

    • 查询一个 根据小标获取元素 get(下标)

    • 查询每一个 集合的遍历

      • 普通的for循环遍历
      • 增强for循环遍历
      • 迭代器遍历（拓展）

异常大家族

族长 Throwable

  • 错误 Error: 代码写的有问题，语法错误，（内存溢出）无限的new出对象

  • 异常 Exception 可以处理，

    • 异常的处理：使用规定的语法，在不影响原来的逻辑情况下，保证程序能够继续运行不崩溃。

      • 编译时期的异常 javac
        • Thread.sleep()
        • 线程相关的异常，io流相关的异常，sql数据库的相关异常
      • 运行时的异常

      异常的处理机制

      虚拟机默认处理方式：

      • 显示异常的类型
      • 显示异常出现的位置
      • 终止当前的线程

      自定义的处理方式：

      1.try-catch代码块

      2.抛出异常 throws

      3.try catch finally

      分支：

      基本数据类型的包装类

      集合遍历的第三组方式，迭代器，

      字符串如何专场整数和浮点数

0614 图形编程

绘制菜单栏 JMenuBar

  • 菜单 JMenu 多个
    • JMenultem多个
  • 案例：
    • 游戏菜单：开始游戏、重新开始、退出游戏
    • 设置：动画开、动画关
    • 帮助：游戏说明、关于
  • 动作接听 ActionListener

监听目标:除了窗口之外，能够产生动作/形态上能够产生变化的组件

在适当位置(Application)创建自定义的监听对象

在适当位置（Application）给某些组件安装动作监听

为不同的组件设置不同的指令（字符串）

图形绘制

前提：要自定义面板

绘制组件：paintComponent()

  • 重写此方法后，虚拟机后独立开一条线程，不断调用此方法
  • 不要创建对象！！！
  • 不要此方法中写修改数据的代码

必备条件

  • 第一含必须保留，条用父类的此方法 super.paintComponent(g)
    • 保证所有的子容器都能够显示正常
  • 在此方法的最后最好写上自动更新和主动更新的代码

绘制内容

  • ​ 绘制图形—–矩形

    • 空心矩形
      • g.drawRect(x,y,w,h)
    • 实心矩形
      • g.fillRect(x,y,w,h)
    • 绘制图片
      • 图片类型：jpg png gif
      • 创建一个文件夹专门放图片
      • g.drawImage(图片对象，x,y,w,h,null);
